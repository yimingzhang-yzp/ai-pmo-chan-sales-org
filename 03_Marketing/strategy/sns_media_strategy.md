## 1. 戦略概要

### 目的
AI PMOちゃんを「社員20名以上のチームが、PMO担当者なしでプロジェクトを回せるようにする唯一のAI」としてポジショニングする。SNSを通じて以下の3つを同時に達成する。

| 目的 | 内容 |
|------|------|
| **認知獲得** | 「PMOを初めて持てる」体験としての認知をSMB層に広める |
| **信頼構築** | PMO課題への知見・実績を通じたブランド権威性の確立 |
| **トライアル獲得** | SNS→LP→フリープラン登録→有料転換のPLGフローを設計・最短化する |

### コアメッセージ（5P Promotionより・優先順位順）

| 優先度 | メッセージ | 主な使用媒体 |
|--------|-----------|------------|
| **1位** | 「Slackから出なくていい。だから今度こそ浸透する。」 | X・Instagram・LP |
| **2位** | 「PMOを雇えない会社に、はじめてのPMOを。」 | note・LinkedIn・LP |
| **3位** | 「催促を、AIに任せよう。」 | X・Instagram |
| **4位** | 「PMO専任の月60万円が、月14,980円になる。」 | LP・LinkedIn・業界メディア |

### AI PMOちゃんのSNS戦略上の強み
AI PMOちゃんは**キャラクターを持つB2Bプロダクト**という希少な存在。競合（Asana・Backlog・ClickUp）はすべてロゴ・機能中心の無機質なSNS運用をしており、**親しみやすいキャラクターによる感情的つながり**は最大の差別化要素となる。

## 2. ターゲット解像度（3C分析より）

### セグメント別ターゲット定義

3C分析の結果、社員20名以上を3セグメントに分類する。初期注力はSセグ・Mセグ。

| 属性 | **Sセグ（20〜50名）** | **Mセグ（50〜200名）** | **Lセグ（200名〜）** |
|------|-------------------|-------------------|-----------------|
| 企業像 | スタートアップ・成長期ベンチャー | 成長中堅ベンチャー・中小企業 | 中堅企業 |
| PMO体制 | 専任なし。代表・PM兼務 | 専任PM 1〜2名が掛け持ち | 専任PMO 1〜5名 |
| 意思決定 | 代表・CTO が即決 | 部門長・CTOが承認（1〜4週間） | IT部門審査→役員承認（1〜3ヶ月） |
| SNS主戦場 | X・Instagram | X・note | LinkedIn・業界メディア |
| 購買スピード | 速い（即日〜1週間） | 普通 | 遅い |

**初期注力:** SセグとMセグ。意思決定が速くSNS経由でトライアルに直結しやすいため。

## 3. 媒体選定と優先度

### 優先度マップ

```
優先度A（コア）   ┌──────────────────────────────────────────┐
                  │  X（Twitter）  │  note  │  Instagram    │
                  └──────────────────────────────────────────┘

優先度B（増幅）   ┌──────────────────────────────────────────┐
                  │  YouTube  │  LinkedIn  │  業界メディア   │
                  └──────────────────────────────────────────┘

優先度C（補完）   ┌──────────────────────────────────────────┐
                  │  SaaS比較サイト  │  PMコミュニティ  │  Eight │
                  └──────────────────────────────────────────┘
```

### 優先度の根拠（3C・5P反映）

| 媒体 | 優先度 | 選定理由（3C/5Pとの接続） |
|------|--------|------------------------|
| **X** | A | SセグCEO・MセグPMの主情報収集源。共感→即トライアルの最短経路。3CインサイトII「SNS共感→即トライアル」の主戦場 |
| **note** | A | MセグPMの深読み・稟議資料作成に活用される。SEO流入×リード獲得の核。5P Price「価格比較訴求」を記事で展開できる唯一の媒体 |
| **Instagram** | A | キャラクター資産の主戦場。競合がブルーオーシャン。エンドユーザー（メンバー層）への認知でPLGのバイラルを起こす |
| **YouTube** | B | 購買検討期の「デモを見たい」に対応。Mセグ・Lセグの稟議資料の補完。5P Placeの「評価フェーズ」を支える |
| **LinkedIn** | B | Lセグ意思決定者・パートナー候補への直接リーチ。「PMOを雇えない会社に〜」メッセージをROI文脈で訴求 |
| **業界メディア** | B | Lセグ決裁者が信頼する第三者権威。5P Promotionの「信頼性担保」機能。稟議突破に使われる |
| **SaaS比較サイト** | C | ITreview・Boxilへの掲載。MセグがSaaS選定時に参照する。5P Place「評価フェーズ」の補完 |
| **PMコミュニティ** | C | connpass・スタートアップSlackへの参入。3CインサイトIV「口コミ・紹介設計」の起点 |
| **Eight** | C | 展示会・ウェビナー後のフォローアップ専用。常時運用不要 |

### 4-1. X（旧Twitter）

**役割:** 認知獲得の最前線 × SNS→即トライアルの直接コンバージョン経路

| 項目 | 内容 |
|------|------|
| **主ターゲット** | SセグCEO・CTO（28〜40歳）/ MセグPM・プロジェクトリーダー |
| **投稿頻度** | 毎日1〜2投稿 |
| **最適投稿時間** | 平日 8:00-9:00（通勤）/ 12:00-13:00（昼休み）/ 21:00-22:00（帰宅後） |
| **KPI** | フォロワー数・インプレッション・引用RT数・プロフィール経由LP流入数・フリープラン登録数 |
| **数値目標** | Phase1: フォロワー500 / Phase2: フォロワー2,000 / Phase3: フォロワー5,000 |

#### コンテンツフォーマット（比率）

| フォーマット | 比率 | 内容・狙い |
|-------------|------|-----------|
| **PMOあるある（共感投稿）** | 40% | 問題提起→Slack上のPMOちゃん解決場面を"見せる"。ビフォーアフター型 |
| **知識・Tip投稿** | 25% | 「PM必読」「プロジェクト管理の鉄則」等。保存・RTされやすい教育コンテンツ |
| **キャラクター投稿** | 20% | PMOちゃんのセリフ・Slack会話再現。「にゃん」語尾・絵文字でポップに |
| **時事連携投稿** | 10% | AI・DX・スタートアップのニュースへの反応。トレンドに乗る |
| **機能紹介・お知らせ** | 5% | 新機能・トライアル告知 |

#### コアメッセージ別投稿設計

| メッセージ | 投稿型 | 具体例 |
|-----------|--------|--------|
| 「Slackから出なくていい」 | 差別化型 | 競合4社を並べて「全部専用ツールを開く必要がある。AI PMOちゃんだけ違う」 |
| 「PMOを初めて持てる」 | 認知型 | 「PMO担当がいない会社が日本に40万社ある。全員詰んでいる」 |
| 「催促を任せよう」 | 共感型 | Slack会話再現でPMOちゃんの催促シーンを見せる |

#### トーン・文体ガイドライン
- **問題提起型:** 断定的・リズム感のある箇条書き。「〜だ。」で終わる文体
- **キャラクター型:** Slack会話を再現。「にゃん」語尾・絵文字・強制感ゼロの口調
- **知識型:** 簡潔で実用的。「〇〇する3つの理由」「〜の落とし穴」

#### スレッド投稿テンプレート
```
【1/5】PMOあるある。
プロジェクトが炎上する「2週間前」、必ず予兆がある。
でも誰も言わない。数字にも出ていない。

【2/5】なぜか？…（問題の深掘り）

【3/5】━━━Slack━━━
🐱 PMOちゃん：「〇〇さん、このまま進むと
　期日に3日足りないにゃ〜😱
　今のうちに対策考えようっか？」
━━━━━━

【4/5】実際のユーザーの声…（社会的証明）

【5/5】今すぐ無料で試す→（LP URL）
#PMO #プロジェクト管理 #AI活用
```

## 6. キャラクター活用ガイドライン

AI PMOちゃんのキャラクターは最大の差別化資産。媒体別に活用方法を変え、一貫したキャラクターブランドを構築する。

### キャラクター基本設定（SNS向け）
- **名前:** AI PMOちゃん（あいぴーえむおーちゃん）
- **語尾:** 「にゃん」「だよ〜」「できるよ！」
- **絵文字:** ✨🐱📋⚡🎯 を基本セット

## 7. クロスポスト設計（コンテンツ連携フロー）

コンテンツを1回作って複数媒体で活用する「ワンソース・マルチユース」設計。
v2.0では**SNS→フリープラン登録の直接転換フロー**を全媒体に組み込む。

```
【ネタ発掘】
競合分析 / βユーザーの声 / PMコミュニティでの質問 / トレンドニュース
        ↓
【X】当日発信（問題提起 or Slack会話再現）
  ＋【Instagram】同日（図解・キャラ投稿）
        ↓（どちらもプロフィールから直接フリープラン登録へ）
        ↓（3〜5日後）
【note】深掘り記事化（稟議支援コンテンツ or SEO記事）
  ＋【LinkedIn】note要約版（ROIフレーミングに変換）
        ↓（月次）
【YouTube】動画化（デモ・比較・事例インタビュー）
        ↓（四半期）
【業界メディア】データ蓄積後に寄稿・プレスリリース
```
